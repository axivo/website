# Kured

This repository uses [Kured](https://kured.dev) to ensure the cluster nodes are properly rebooted during a scheduled OS upgrade reboot event.

## Slack

Slack [notifications](https://kured.dev/docs/configuration/#notifications) can be enabled, allowing the end-user to receive notifications when each cluster node is rebooted.

<Steps>
### Slack Application

Create a new [application](https://api.slack.com/apps) from scratch, named `Kured`, and assign it to the intended workspace where Kured notifications will be received.

### Webhook

Activate [incoming webhooks](https://api.slack.com/messaging/webhooks) and create a new webhook, specifying in what channel Kured will have permissions to post cluster updates. The webhook will have the following format:

```shell
https://hooks.slack.com/services/<token>/<token>/<token>
```

### Notify URL

Encrypt the `global_map.credentials.kured.slack.notify.url` value with [`ansible-vault`](/k3s-cluster/tutorials/configuration/ansible/#vault) and insert it into the <Var code href={`https://${repository.home}/blob/${repository.tag}/inventory/cluster/group_vars/all.yaml`}>all.yaml</Var> group variables file. Notify URL decrypted format:

```yaml
global_map:
  credentials:
    kured:
      slack:
        notify:
          url: slack://<token>/<token>/<token>
```

</Steps>
