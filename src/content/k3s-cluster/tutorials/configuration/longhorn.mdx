# Longhorn

This repository uses [Longhorn](https://longhorn.io), delivering simplified, cloud-native persistent block storage without the cost overhead of open core or proprietary alternatives.

## Backup

Longhorn uses various backup target endpoints to access a `backupstore`. A backup target endpoint with `cifs` protocol is configured by default, used with an existing TrueNAS Scale `SMB` share and related <Var code href={`https://${repository.home}/blob/${repository.tag}/roles/longhorn/templates/credentials.j2`}>credentials.j2</Var> secret:

```yaml
longhorn_vars:
  kubernetes:
    backup:
      enabled: true
      target: cifs://nas.noty.cc/backup
```

Review the official [documentation](https://longhorn.io/docs/latest/snapshots-and-backups/backup-and-restore/set-backup-target) to configure the backup target endpoint with a different protocol. Depending on the chosen protocol, the <Var code href={`https://${repository.home}/blob/${repository.tag}/roles/longhorn/templates/credentials.j2`}>credentials.j2</Var> secret will require an update.

## Node Drain

The node drain behavior is controlled with the `longhorn_vars.kubernetes.default_settings.node_drain_policy` and `longhorn_vars.kubernetes.default_settings.pod_deletion_policy` variables, defined in the <Var code href={`https://${repository.home}/blob/${repository.tag}/roles/longhorn/defaults/main.yaml`}>main.yaml</Var> defaults file.
